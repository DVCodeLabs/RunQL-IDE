# Getting Started with RunQL

RunQL is a SQL editor built on top of open source Code OSS / VSCodium foundations. It is designed for database work, documentation, saved queries, schema context, and AI-assisted workflows.

## Installation

Download the latest desktop build from:

- [RunQL Releases](https://github.com/DVCodeLabs/RunQL-IDE/releases)

Learn more about the open source project here:

- [RunQL Open Source](https://runql.com/opensource/)

## First Steps

After installing RunQL:

1. Open your project folder.
2. Open the built-in RunQL welcome page.
3. Initialize the workspace if prompted.
4. Connect to your database.
5. Start saving queries, schema docs, and ERD artifacts into the workspace.

## Basic Usage

RunQL keeps database context in files so that:

- your team can review database artifacts in Git
- AI tools can work from real schema and query context
- saved queries are searchable and reusable

By default, RunQL uses:

- Open VSX for extensions
- telemetry-disabled desktop builds
- the bundled RunQL extension in the desktop app releases

## Keyboard Shortcuts

Useful defaults:

- `Ctrl+P` / `Cmd+P`: quick open
- `Ctrl+Shift+P` / `Cmd+Shift+P`: command palette
- `Ctrl+,` / `Cmd+,`: settings
- `Ctrl+K Ctrl+S` / `Cmd+K Cmd+S`: keyboard shortcuts
